## 設計原則

### 以「資料項目」為主（推薦）

```
共用資料表（基礎資料）
├── stocks（股票基本資料）
├── daily_prices（每日股價）    ← 所有功能共用
└── institutional_trading（法人）← 所有功能共用

功能專用表（特有資料）
├── themes（題材）             ← 題材追漲專用
├── theme_stocks（題材股票關聯）
└── breakout_signals（突破訊號）← 箱型突破專用
```

### 設計邏輯

| 比較 | 以功能為主 | 以資料為主 |
|------|-----------|-----------|
| 資料重複 | ❌ 多 | ✅ 少 |
| 擴展性 | ❌ 差 | ✅ 好 |
| 需要 JOIN | 不需要 | ✅ 需要 |
| **建議** | ❌ | ✅ **推薦** |

> **注意**：以資料為主會需要 JOIN 多張表，但只要有索引就很快

## 資料表設計

### 1. stocks（股票基本資料）

```sql
CREATE TABLE stocks (
  stock_id VARCHAR(10) PRIMARY KEY,
  name VARCHAR(50),
  market VARCHAR(10),        -- TSE / OTC
  industry VARCHAR(50),
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 2. daily_prices（每日股價）

```sql
CREATE TABLE daily_prices (
  id SERIAL PRIMARY KEY,
  stock_id VARCHAR(10) REFERENCES stocks(stock_id),
  date DATE NOT NULL,
  open DECIMAL(10,2),
  high DECIMAL(10,2),
  low DECIMAL(10,2),
  close DECIMAL(10,2),
  volume BIGINT,
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(stock_id, date)
);

CREATE INDEX idx_daily_prices_stock_date ON daily_prices(stock_id, date DESC);
```

### 3. institutional_trading（法人買賣超）

```sql
CREATE TABLE institutional_trading (
  id SERIAL PRIMARY KEY,
  stock_id VARCHAR(10) REFERENCES stocks(stock_id),
  date DATE NOT NULL,
  foreign_buy INTEGER,       -- 外資買賣超（張）
  trust_buy INTEGER,         -- 投信買賣超
  dealer_buy INTEGER,        -- 自營商買賣超
  created_at TIMESTAMP DEFAULT NOW(),
  UNIQUE(stock_id, date)
);

CREATE INDEX idx_institutional_stock_date ON institutional_trading(stock_id, date DESC);
```

### 4. themes（題材）

```sql
CREATE TABLE themes (
  id SERIAL PRIMARY KEY,
  name VARCHAR(100),
  description TEXT,
  theme_date DATE,
  strength INTEGER,          -- 強度 1-5
  duration VARCHAR(20),      -- short/medium/long
  is_confirmed BOOLEAN,
  created_at TIMESTAMP DEFAULT NOW()
);
```

### 5. theme_stocks（題材與股票關聯）

```sql
CREATE TABLE theme_stocks (
  id SERIAL PRIMARY KEY,
  theme_id INTEGER REFERENCES themes(id),
  stock_id VARCHAR(10) REFERENCES stocks(stock_id),
  impact_ratio INTEGER,      -- 受惠比例 %
  notes TEXT,
  created_at TIMESTAMP DEFAULT NOW()
);
```

## 資料流程

```
證交所 / Yahoo Finance API
         │
         ▼ (每日盤後抓取)
      後端程式
         │
         ▼ (儲存)
    Supabase 資料庫
         │
         ▼ (讀取)
      前端網站
```

### 哪些資料存 DB？

| 資料 | 來源 | 頻率 | 存入 DB |
|------|------|:----:|:-------:|
| 每日股價 | 證交所 API | 每日盤後 | ✅ |
| 三大法人買賣超 | 證交所 API | 每日 20:00 後 | ✅ |
| 股票基本資料 | 證交所 | 一次性 | ✅ |
| **即時股價** | 證交所/Yahoo | 即時呼叫 | ❌ 不存 |

### 兩種資料處理方式

| 類型 | 做法 |
|------|------|
| **歷史資料** | API → 存 DB → 功能讀取 DB |
| **即時資料** | API → 直接顯示（不存 DB）|

## 前端串接範例

```javascript
import { createClient } from '@supabase/supabase-js'

const supabase = createClient(SUPABASE_URL, SUPABASE_KEY)

// 取得某股票近 20 日法人買賣超
async function getInstitutionalData(stockId, days = 20) {
  const { data } = await supabase
    .from('institutional_trading')
    .select('*')
    .eq('stock_id', stockId)
    .order('date', { ascending: false })
    .limit(days)
  
  return data
}
```
